# 需求整理

## 目标

建设一个报告生成工具：输入已有监测数据和报告大纲，输出可读的可视化报告。报告应包含基础信息、题目统计图表、分学校/区域等维度对比，以及基于统计规则生成的文字解读。

## 已确认的数据形态

### 分题 sheet 型 Excel

地理教师样例属于这种结构：

- `首页`：项目名称、起止时间、表单标题、区域维度、学期维度、学校数量、用户数量等基础信息。
- `题型统计`：每道题的题型，例如单选题、多选题、填空题、量表题、表格组合题。
- `字段映射关系`：题号、题 id、字段 id、字段名称、字段标题。
- `维度映射关系`：题号、题 id、维度 id、维度名称、维度组名称、维度标题。
- 数字 sheet：sheet 名为题号，如 `1`、`2`、`16`。每行通常表示某个用户在某个字段/选项上的取值。

题号 sheet 的关键列包括：

- `问题id`
- `字段id`
- `字段名称`
- `字段取值`
- `维度id`
- `维度名称`
- `学科维度`
- `年级维度`
- `学期维度`
- `学校维度`
- `区域维度`
- `用户维度`
- `状态`

### 普通宽表型数据

文档还要求兼容普通数据结构：第一行为题目信息，之后每一行是一个用户的全部信息。该结构通常适合先转成统一中间表，再进入相同统计和图表流程。

## 第一版应支持的输出

- 基础信息表/作答事实表：保留用户 id、学校、学科、学期、年级、区域、题号、字段、字段取值、来源文件、来源 sheet、来源行号等，用作所有交叉分析和复核的底表。
- 题目表：记录题号、题型、题 id、字段 id、字段名称、字段标题、维度映射、选项顺序、连接键。
- 数据重构规则表：记录公式、赋值、题目组合、指标重构等数据层规则。
- 重构后指标事实表：数据重构规则稳定后生成，便于后续生成、复核和跨图表复用。
- 报告生成规则表：记录章节、测试图类型、筛选范围、对比范围、写作指标和写作要求。
- 派生作答数量表：每道题每个学校的作答人数，作为预览或缓存，不作为分析底表。
- 派生选项汇总表：每道题各选项/字段的选择数或数值汇总，作为预览或缓存，不作为分析底表。
- 简易图表预览：例如题型分布、某题按学校的柱状图、某题选项汇总柱状图。
- 后续报告写作输入：供“章节大纲 + 题号 + 图表样式 + 文字规则”驱动。

## 交叉分析需求

后续报告会包含大量交叉分析，因此中间表不能只保存单题汇总结果。需要支持：

- 学校内某单题选项比例柱状图，例如某学校内 Q1 学历分布。
- 多选题学校 x 选项热力图，例如 Q41 不同学校对各面批要求选项的选择比例。
- 多题交叉分析，例如按 `user_id` 连接 Q1 与 Q16，比较不同学历教师对教材内容科学性的评价分布。
- 部分数据报告，例如某地区或某类学校与全体均值、比例、分布的对比。
- 结果复核定位，任何汇总值都能回到 `source_file + source_sheet + source_row_index + user_id + question_no + field_id`。

## 特殊规则拆分需求

原“特殊处理表”需要拆成两个部分。

### 数据重构规则

数据重构规则作用在原始数据和第一类事实表之上，用于生成稳定的中间结果。典型规则包括：

- 题目赋值，例如默认单选题为 0/1，但某道题按 0.2 计分。
- 量表赋值，例如“非常不符合=1，不符合=2，符合=3，非常符合=4”。
- 多题组合，例如几道题按权重合成一个综合指标。
- 选项合并或重命名。
- 矩阵题或表格组合题展开。
- 填空题数值化或分组。

这部分规则稳定后，需要生成 `reconstructed_indicator_fact` 中间表。该表应保留重构规则 id、指标 id、用户 id、学校、指标值、来源题号、来源事实行和复核说明，方便后续生成与核对。

### 报告生成规则

报告生成规则服务实际报告框架。用户可以先选择某些章节进行测试，指定图表类型，并修改写作指标。典型内容包括：

- 章节和小节。
- 使用哪些题号或重构指标。
- 图表类型，例如柱状图、堆叠柱状图、热力图、箱线图。
- 筛选范围，例如某些学校、区域、学段或样本范围。
- 对比范围，例如学校 vs 全体、部分数据 vs 全体。
- 写作指标，例如均值、比例、极差、众数、分布、差值百分点。
- 写作要求，例如重点描述最高项、最低项、差异较大项。

## 当前样例观察

地理教师样例的首页显示：

- 表单标题：地理教师课程实施与教材使用情况表。
- 学期：2024~2025学年第二学期。
- 项目下挂学校数量：9。
- 填写表单的学校数量：9。
- 填写表单的用户数量：52。

题型统计显示：

- 单选题：27。
- 填空题：12。
- 多选题：13。
- 量表题：4。
- 表格组合题：1。

抽样题目：

- Q1：教师学历，4 个字段，带年级维度映射。
- Q2：教师职称，6 个字段。
- Q16：教材内容科学性，4 个量表选项。
- Q41：学校面批要求，6 个多选字段。

## 待确认问题

- 是否所有分题 sheet 均可默认用 `用户维度` 作为作答人唯一标识。
- 第一类表字段范围是否由用户提供说明来确定，哪些学校性质、学校数量、学科、年级、学期等字段必须保留。
- 第一类表和第二类题目表中的重叠字段如何处理：事实表是否保留可读标签，还是只保留连接键并全部通过题目表解释。
- 数据重构规则由谁维护：用户填写配置、系统自动推断，还是先由人工确认后固化。
- 报告生成规则的输入形式：用户提供 Excel/CSV 大纲、Markdown 大纲，还是交互式选择章节和图表类型。
- 题目没有稳定题干，且题号跨表不唯一；后续必须使用 `question_id` 作为全局题目标识。
- 普通宽表数据是否会包含学校、区域、学期、用户等固定列名，还是需要单独字段映射。
- 填空题是否需要分为“数值填空”和“文本填空”两类处理。
- 报告文字是否需要固定模板风格，还是根据图表类型动态生成。
